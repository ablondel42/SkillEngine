# Quality Checklist

Quick sanity check before delivering your design system.

---

## Before You Start

- [ ] Asked all 5 interview questions
- [ ] Understand the domain and can name 5-10 domain components
- [ ] Tech stack is clear (framework, styling, libraries)
- [ ] Know which screens to build as examples

---

## Tokens

- [ ] Primary palette has full range (50-900)
- [ ] Neutral palette is complete
- [ ] Semantic colors defined (success, warning, error, info)
- [ ] Typography scale (sizes, weights, line heights)
- [ ] Spacing scale (based on 4px)
- [ ] Border radius values
- [ ] Shadow levels

---

## Components

**Core:**
- [ ] Button (all variants + sizes + states)
- [ ] Input (all states + error handling)
- [ ] Card
- [ ] Alert (all variants)
- [ ] Badge (all variants)

**Domain-specific:**
- [ ] 5-10 components tailored to the app's use cases

---

## Examples

- [ ] Showcase page with all tokens + components
- [ ] 2+ context pages showing components in realistic scenes

---

## Documentation

- [ ] guidelines.md - tokens, components, usage
- [ ] accessibility.md - WCAG guidelines, testing tips

---

## Quick Tests

- [ ] Keyboard navigation works
- [ ] Focus indicators visible
- [ ] Color contrast passes (4.5:1 for text)
- [ ] Dark mode renders correctly (if applicable)
- [ ] No console errors

---

## Before Delivering

Skim through and ask:
- Does this feel complete and coherent?
- Would I be confident showing this to a client?
- Are the example pages visually appealing?

If yes, you're ready to deliver.
